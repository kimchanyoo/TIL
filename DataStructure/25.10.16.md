# 그래프(Graph)

## 비선형 구조
* 데이터를 저장하기 위한 방법으로 데이터 간의 관계를 이루면서 '계층적인 구조'를 가짐
* **'일렬로 나열되지 않은 자료구조'** 형태
    * 계층적으로 데이터의 관계가 부모-자식 관계, 연결 관계, 또는 소속 관계 등을 가지고 있어서 계층적이거나 상호 연결

## 그래프
* 정점과 그들 사이를 연결하는 간선으로 표현된 자료구조
    * 각 정점은 데이터를 저장
        * Node라고도 부름
    * 간선은 정점들 간의 관계
* 모든 그래프는 공식적으로 `정점(V)`와 `간선(E)`으로 그래프는 `G = {V, E}`로 표현

<img width="771" height="387" alt="Image" src="https://github.com/user-attachments/assets/18e9bf89-ad57-45a0-8e69-ff3bf61b9b67" />

### 특징
1. **정점(Vertex)과 간선(Edge)**
2. **방향성(Directedness)**
    * 방향성이 있는 그래프에서는 간선에 방향이 있음
        * 한 정점에서 다른 정점으로만 이동 가능
    * 방향성이 없는 그래프에서는 간선에 방향이 없음
        * 양방향으로 움직일 수 있음
3. **가중치**
    * 그래프의 간선은 가중치를 가질 수 있음
    * 두 정점 사이의 거리, 비용, 우선 순위 등을 나타냄
4. **사이클**
    * 그래프에서 사이클은 한 정점에서 시작해서 다시 그 정점으로 돌아올 수 있는 경로
    * 사이클이 없는 그래프를 `트리`라고도 함
5. **연결성**
    * 그래프에서 모든 정점이 연결되어 있는지 나타내는 개념
    * 연결되어져 있으면 연결 그래프
    * 연결되어져 있지 않으면 비연결 그래프
6. **차수**
    * 각 정점에 연결된 간선의 개수
    * 정점의 중요성이나 연결성을 나타내는 지표

### 표현 예시
1. **인접 행렬**
    * 그래프의 연결 상태를 2차원 배열로 나타내는 자료구조
    * 노드의 개수를 `N`이라고 할 때, `N x N` 크기의 배열을 사용하여 그래프의 각 노드 간 연결 여부를 표현
    * 그래프의 연결 관계를 0 또는 1로 구현
        * 만약 노드 간에 연결이 있다면 1을 나타내고 연결이 없으면 0으로 나타냄
    * 노드 간의 연결 관계를 상수 시간에 확인할 수 있으며, 간선의 개수에 비례하는 메모리 공간을 요구

```
무방향 그래프(Undirected Graph) -> 인접 행렬(Adjacency Matrix)

- [1,1] ‘0’의 값을 기준으로 ‘0’은 연결이 되어있지 않기에 0의 값을 가집니다.
- [1,2] ‘0’의 값을 기준으로 ‘1’은 연결되어 있기에 1의 값을 가집니다.
- [1,3] ‘0’의 값을 기준으로 ‘2’는 연결되어 있기에 1의 값을 가집니다.

- [2,1] ‘1’의 값을 기준으로 ‘0’은 연결되어 있기에 1의 값을 가집니다.
- [2,2] ‘1’의 값을 기준으로 ‘1’은 연결이 되어 있지 않기에 0의 값을 가집니다.
- [2,3] ‘0’의 값을 기준으로 ‘2’는 연결되어 있기에 1의 값을 가집니다.

- [3,1] ‘2’의 값을 기준으로 ‘0’은 연결되어 있기에 1의 값을 가집니다
- [3,2] ‘2’의 값을 기준으로 ‘1’은 연결되어 있기에 1의 값을 가집니다
- [3,3] ‘2’의 값을 기준으로 ‘2’은 연결이 되어 있지 않기에 0의 값을 가집니다.
```

```
방향 그래프(Directed Graph) -> 인접 행렬(Adjacency Matrix)

- [1,1] ‘0’의 값을 기준으로 ‘0’은 연결이 되어있지 않기에 0의 값을 가집니다.
- [1,2] ‘0’의 값을 기준으로 ‘1’은 방향성이 존재하지 않기에 0의 값을 가집니다.
- [1,3] ‘0’의 값을 기준으로 ‘2’는 방향성이 존재하지 않기에 0의 값을 가집니다.

- [2,1] ‘1’의 값을 기준으로 ‘0’은 방향성이 존재하므로 1의 값을 가집니다
- [2,2] ‘1’의 값을 기준으로 ‘1’은 연결이 되어 있지 않기에 0의 값을 가집니다.
- [2,3] ‘0’의 값을 기준으로 ‘2’는 방향성이 존재하므로 1의 값을 가집니다.

- [3,1] ‘2’의 값을 기준으로 ‘0’은 방향성이 존재하므로 1의 값을 가집니다.
- [3,2] ‘2’의 값을 기준으로 ‘1’은 방향성이 존재하지 않기에 0의 값을 가집니다
- [3,3] ‘2’의 값을 기준으로 ‘2’은 연결이 되어 있지 않기에 0의 값을 가집니다.
```

2. **인접 목록**
    * 그래프의 연결 상태를 연결 리스트로 나타내는 자료구조
    * 그래프의 희소성을 고려하여 메모리를 효율적으로 사용
    * 노드 간의 연결 관계를 확인하기 위해서는 인접한 노드들을 순회해야 하므로, 인접 행렬에 비해 상대적으로 더 많은 시간이 소요

```
무방향 그래프(Undirected Graph) -> 인접 목록(Adjacency list)

- ‘0’의 값을 기준으로 0 → 1 → 2까지 연결 리스트가 구성되었습니다.
- ‘1’의 값을 기준으로 1 → 0 → 2까지 연결 리스트가 구성되었습니다.
- ‘2’의 값을 기준으로 2 → 0 → 1까지 연결 리스트가 구성되었습니다.
```

<img width="731" height="341" alt="Image" src="https://github.com/user-attachments/assets/e25bbf13-a156-4e57-96ba-a65226a2dc9d" />


```
방향 그래프(Directed Graph) -> 인접 목록(Adjacency list)

- ‘0’의 값을 기준으로 방향이 존재하지 않기에 0의 값만 표현합니다.
- ‘1’의 값을 기준으로 1 → 0 → 2까지 연결 리스트가 구성되었습니다.
- ‘2’의 값을 기준으로 2 → 0 까지 연결 리스트가 구성되었습니다.
```

## 그래프의 종류

<img width="1100" height="594" alt="Image" src="https://github.com/user-attachments/assets/36615f53-0b5f-4950-aaae-b0e273d9077e" />


### 무방향 그래프
* 간선에 '방향이 없는 그래프'로 노드들이 서로 연결됨
* 노드들이 서로 연결되어 있으면 간선은 양방향으로 표현

<img width="261" height="292" alt="Image" src="https://github.com/user-attachments/assets/71004473-4156-4e1e-8cb5-b3dc14710502" />

**사용 예시**
* 친구 관계나 네트워크 연결

**응용 알고리즘**
* **깊이 우선 탐색 (DFS) 알고리즘**
* **너비 우선 탐색 (BFS) 알고리즘**
* **최소 신장 트리 (MST)**

### 방향 그래프
* 간선에 ‘방향이 있는 그래프’로 간선은 출발 노드와 도착 노드를 가리킴
* 각각의 노드가 서로를 향해 방향을 가리키는 그래프
* 한 노드에서 다른 노드로 갈 수 있는 경로가 있을때만 그 노드들은 방향 그래프에서 연결되어 있다고 볼 수 있음

<img width="239" height="307" alt="Image" src="https://github.com/user-attachments/assets/656ff298-819e-4dc3-94ce-44f90296bcec" />

**사용 예시**
* 인터넷의 웹 페이지 간의 링크, 도로 네트워크의 트래픽 흐름, 의사 결정 트리 등을 방향 그래프로 표현
* 최단 경로 알고리즘, 흐름 네트워크 문제, 예산 할당 등의 문제를 해결하는 데에 활용

**응용 알고리즘**
* **깊이 우선 탐색 (DFS) 알고리즘**
* **너비 우선 탐색 (BFS) 알고리즘**
* **위상 정렬 알고리즘**
    * 방향 그래프의 정점들을 일렬로 나열하는 알고리즘
    * 사이클이 없는 방향 그래프에서만 적용 가능

### 가중 그래프
* 간선에 ‘가중치가 할당된 그래프’로 가중치는 노드 간의 관계나 강도, 비용, 거리 등을 나타냄

<img width="447" height="275" alt="Image" src="https://github.com/user-attachments/assets/c833f878-c6ae-4cf4-b4df-b1e608a13fb9" />

**사용 예시**
* 최단 경로 알고리즘, 흐름 네트워크 문제, 예산 할당, 회로 설계 등 다양한 문제를 해결하는 데에 활용

**응용 알고리즘**
* **최단 경로 (Shortest Path) 알고리즘**
    * 그래프에서 두 정점 사이의 가장 짧은 경로를 찾는 알고리즘
* **최대 유량 (Maximum Flow) 알고리즘**
    * 그래프에서 한 정점에서 다른 정점으로의 최대 유량을 찾는 알고리즘

### 연결 그래프
* 그래프에서 ‘모든 노드가 최소한 하나의 경로로 연결’된 그래프
* 어떤 두 노드 사이에 경로가 존재하면 그 노드들은 연결 그래프에서 연결

<img width="270" height="260" alt="Image" src="https://github.com/user-attachments/assets/c119c614-9959-4ef8-b76a-efbecd77b711" />

**사용 예시**
* 인터넷의 웹 페이지 간의 링크 관계, 도로 네트워크의 연결성, 사회적 관계 네트워크 등 다양한 문제를 해결하는데 활용

**응용 알고리즘**
* **연결성 확인 (Connectivity Check) 알고리즘**
    * 그래프 내에서 두 정점 사이의 연결성을 확인하는 알고리즘
* **오일러 경로 (Eulerian Path) 알고리즘**
    * 그래프의 모든 간선을 한 번씩만 지나는 경로를 찾는 알고리즘

### 비연결 그래프
* 그래프에서 ‘일부 노드가 다른 노드와 연결되지 않은 그래프’
* 하나 이상의 연결 요소로 이루어져 있으며 각 연결 요소는 자체적으로는 연결되어 있지만 다른 연결 요소와는 연결되어 있지 않음

<img width="230" height="285" alt="Image" src="https://github.com/user-attachments/assets/bb4452f2-205f-4051-98d9-8e850052e490" />

**사용 예시**
* 도로 네트워크에서 도로가 각각의 도시를 연결하고 있지만 도시들 간에는 연결이 되어 있지 않은 경우 비연결 그래프로 표현

**응용 알고리즘**
* **연결성 확인 (Connectivity Check) 알고리즘**
    * 그래프 내에서 두 정점 사이의 연결성을 확인하는 알고리즘
* **오일러 경로 (Eulerian Path) 알고리즘**
    * 그래프의 모든 간선을 한 번씩만 지나는 경로를 찾는 알고리즘
* **최소 신장 트리 (MST)**

### 순환 그래프
* 그래프에 ‘사이클이 존재하는 그래프’로 한 노드에서 시작하여 다시 해당 노드로 돌아올 수 있는 경로가 있는 그래프
    * 한 개 노드에서 시작하여 다른 노드들을 거쳐 다시 ‘자기 자신으로 돌아오는 경로’를 가지는 그래프

<img width="450" height="280" alt="Image" src="https://github.com/user-attachments/assets/19396fe7-0f36-4034-8248-6f02f02659de" />

**사용 예시**
* 전기 회로에서의 신호의 흐름, 사이클링 알고리즘, 경로 최적화 등을 순환 그래프로 표현

**응용 알고리즘**
* **사이클 검사 (Cycle Detection)**
    * 그래프 내에 순환 구조가 있는지 확인하는 알고리즘
* **위상 정렬 (Topological Sort)**
    * 순환 그래프에서는 적용할 수 없지만, 사이클이 없는 방향 그래프에서 정점들을 일렬로 나열하는 알고리즘

### 비순환 그래프
* 그래프에 ‘사이클이 존재하지 않는 그래프’로 한 노드에서 시작하여 해당 노드로 돌아오는 경로가 없는 그래프
    * 어떠한 경로도 자기 자신으로 돌아오지 않는 순환이 없는 그래프

<img width="351" height="336" alt="Image" src="https://github.com/user-attachments/assets/d68b61fe-fe26-41f2-862f-50c782ddbfcd" />

**사용 예시**
* 전기 회로에서의 신호의 흐름, 사이클링 알고리즘, 경로 최적화 등을 순환 그래프로 표현

**응용 알고리즘**
* **최단 경로 (Shortest Path) 알고리즘**
* **위상 정렬 (Topological Sort)**
    * 순환 그래프에서는 적용할 수 없지만, 사이클이 없는 방향 그래프에서 정점들을 일렬로 나열하는 알고리즘
